## 📋 子领域定位

协作模型定义了团队成员之间的工作方式和互动模式。在分布式团队和远程工作成为常态的今天，高效的协作模型对于团队生产力和员工满意度至关重要。

本领域涵盖远程工作流（异步沟通、时区管理、远程文化建设）、知识共享（结对编程、集体 Ownership、技术分享）和团队拓扑（流对齐团队、平台团队、复杂子系统团队）三大方向。

## 🗂️ 专项列表

### [C01. Remote_Workflows](C01_Remote_Workflows/README.md)

远程工作流确保分布式团队高效协作。本专项详解异步沟通原则（文档优先、减少会议）、远程会议最佳实践、时区管理策略、以及远程文化建设。涵盖数字游民政策、远程办公工具和异步决策流程。

### [C02. Knowledge_Sharing](C02_Knowledge_Sharing/README.md)

知识共享防止信息孤岛和单点故障。本专项覆盖结对编程技术、Mob Programming 集体协作、内部技术分享（午餐学习、技术雷达）、以及文档驱动文化。涵盖知识管理工具和导师制度设计。

### [C03. Team_Topologies](C03_Team_Topologies/README.md)

团队拓扑提供组织结构设计的系统方法。本专项详解流对齐团队（Stream-aligned Team）、平台团队（Platform Team）、复杂子系统团队（Complicated Subsystem Team）和赋能团队（Enabling Team）四大拓扑类型。涵盖团队交互模式（协作、服务、促进）和康威定律应用。

## 💼 实践案例

### 异步沟通指南

**原则**:
- 文档优先：决策必须记录在案
- 减少同步会议：会议是最后手段
- 透明公开：默认公开，按需私密
- 尊重时区：不期望即时回复

**工具组合**:
| 场景 | 工具 | 响应期望 |
|------|------|----------|
| 紧急事务 | 电话/短信 | 立即 |
| 快速问答 | 即时消息 | 4小时内 |
| 详细讨论 | 邮件/文档 | 24小时内 |
| 知识沉淀 | Wiki/Notion | 无需响应 |
